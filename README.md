      <h4>
        Instructions:
      </h4>
        <ol>
          <li>
            This process is case, character and whitespace sensitive. Any of the wrong input  can lead to complete termination of the process.
          </li>
          <li>
            The recommended method to interact with this process is to copy option from given list of options and paste it to use. 
          </li>
          <li>
            To avoid horizontal scroll, before reading code set font size <b>11</b> for <b>Dev-C++ 5.11 IDE</b>.  
          </li>
          <li>
            Maximum range of above water level can be 2.59 billion kms i.e. 25900000000 kms and range of below water level can be maximum 25 thousand kms i.e. 
            25000 kms. 
          </li>
          <li>
            Entering distance between sender and receiver device 0 is invalid. As distance is equal to 0 means no gap/distance between them. As there is always possibility 
            of distance such as 0.001km i.e. 1m, 0.00001km i.e. 1cm, 0.0000001km i.e. 0.1mm or 0.000000000001km i.e. 1nm etc.
          </li>
          <li>
            Writing <b>Display info</b> i.e. whitespace between "<b>Display</b>" and "<b>info</b>" will terminate the process by calling <b>Invalid_Statement()</b>
            function. As <b>Dev-C++ IDE Linker</b> (Front-hand executeable file) does not accept whitespace as input. Correct will be <b>Display-info</b> (as 
            given in the options).
          </li>
          <li>
            Writing distance such as --10, -(-10), -1(-10), 2(500-200), (12,000) etc. all will be invalid input. As <b>IDE</b> considers each part as individual 
            character. Such as --(10), here -- as two minus signs without any operand, -1(-10), two inputs, for 2(500-200) it may not show error but 
            funalitionalities will give unexpected output such as 2(500-200), it'll show distance 2km which is wrong output as calculation shows 600km and 
            <b>Networking Path</b> selected will also be shown wrong as distance accepted as input is wrong. Same is if enter 12,000 it'll consider it 12km. So, 
            for accurate output enter distance in <b>integer</b> or <b>floating point
            </b> etc. but without expression and commas etc.
          </li>
        </ol>
